Modules — the whole vision

M1 — Configurator + ingestion Partial

Faithful read-only Configurator clone; ingest any project context.

  • Configurator view (tree, grouped settings, command tree)
  • Upload any .bos → parsed & rendered
  • Per-project store (SQLite snapshots)
  • Milesight gateway config → LoRa/IoT topology
  • Add PDF/Word/any file → indexed context
  • Proactively suggest missing context

M2 — Agent: "how do I implement X" Partial

A teacher, not just a doer — grounded in the loaded system.

  • Agent chat dock
  • Pluggable runtime (Claude Code → Codex → OpenRouter)
  • State what + pedagogical steps + finished code
  • Emit runnable playbook JSON
  • Route emitted playbook into the auto-grader
  • RAG over context (grounds on .bos; embeddings to come)
  • Auto-select MCP/skills by context

M3 — Code standard To do

Define, teach, and enforce a project code standard retroactively.

  • Define / recommend a standard → wiki canonical page
  • Standards audit via ECC reviewers
  • List each flaw + show the fix diff

M4 — Themeable wiki vault Partial

Auto-documented, pedagogical, exportable, re-themeable.

  • Per-feature wiki docs (built as we go)
  • Token-based theme (:root)
  • This live progress dashboard
  • Theme switcher + editable pages
  • Network address/login registry (vault-backed)
  • 10 HTML-effectiveness artifacts; static-site export

M5 — Memory + instincts + roster Built

Persistent per-project memory; standing instincts; an agent roster.

  • Agent instincts (system prompt)
  • Per-project persistent memory
  • Multi-agent roster + share protocol
  • Instincts persistence + versioning

M6 — Enterprise security Partial

Encrypted vault, Microsoft SSO unlock, security scans.

  • AES-256-GCM vault, key in OS keychain
  • Microsoft Entra SSO gates unlock
  • Plaintext-secret guardrail
  • ECC agents-shield security scans
  • Enterprise-criteria gate before "done"

M7 — Knowledge hunger To do

Crawl official docs + forums; propose improvements.

  • Crawler sidecar (docs / knowledgebase)
  • Track ComfortClick + IoT forums
  • Prioritise bOS Configurator/Service/Client
  • Eureka → prompt, make the case, show how

M8 — Version control (GitHub × ComfortClick) To do

Semantic .bos diffs, human PRs with AI docs, timeline.

  • Snapshot semantic diff engine
  • In-app PRs (human narrative + AI docs)
  • Comment/argue threads + merge
  • Project timeline

M9 — Runtime/harness & ECC Partial

Adapter detection; ECC as the coding harness.

  • Detect Claude Code → Codex → OpenRouter
  • Prefer local runtimes
  • ECC agents drive the dev flow (reviewers, fixers, workflows)
  • CI gate + community health (public repo)
  • Auto-select MCP/skills per file type
  • DesignSync for professional client UI
