Code Standard & Audit Built

Hyperion has one canonical way to write its code, derived from the conventions the codebase already follows — and a deterministic audit that flags anything that drifts away from it, with the fix spelled out. No linter daemon, no network: pure heuristics over the source text.

The Rust standard

The backend (src-tauri/src) is Rust edition 2021. Two tools are the source of truth for style and lint, and both must be clean before anything merges:

  • Formatting — cargo fmt --all; CI runs cargo fmt --all -- --check. Spaces, never tabs.
  • Lint — cargo clippy --all-targets -- -D warnings. Warnings are errors.
  • No panics in non-test code — no .unwrap(), no .expect(…), no panic!/todo!/unimplemented!/unreachable!. The app should never crash on bad input; surface the condition instead.
  • Errors as Result<_, String> at the command layer — every #[tauri::command] returns either a value or a human-readable String error, and lower layers propagate with ?. Lock poisoning is recovered (unwrap_or_else(|e| e.into_inner())), never unwrapped.
  • One #[cfg(test)] module per file — tests live in a single trailing mod tests block. Inside tests, .unwrap() is fine and expected.
  • Module doc-comment — each file opens with a // header explaining what it does and its read-only-toward-bOS contract.

The TypeScript standard

The webview (src/main.ts) is a vanilla, dependency-light TypeScript port of the original Flask view. It carries a deliberate // @ts-nocheck at the top (it is a faithful port, not a rewrite); new, strictly-typed logic is extracted into small src/lib helpers as it grows. The baseline:

  • No stray console.log(…) — debug logging doesn't ship.
  • Spaces, not tabs, and a single trailing newline — same as the Rust side.
  • Talk to the backend through invoke() — the command shim, never fetch.

The audit

The standard is enforced by a pure function in standard.rs:

audit(files: &[(path, contents)]) ──► Vec<Finding { path, line, rule, message, severity }>

It takes file contents as input — no filesystem walk, no network — so it is fully unit-testable with synthetic source. The code_audit command is the only piece that reads the real sources off disk (src-tauri/src/*.rs + src/*.ts) and hands them to audit. Each rule is a low-noise text heuristic; comment lines that merely mention a pattern (like this page's own backend) are exempt.

RuleApplies toSeverityFlags & suggested fix
no-unwrapRustmedium .unwrap() outside a #[cfg(test)] region — propagate with ? or map to Result<_, String>.
no-expectRustmedium .expect(…) in non-test code — return a descriptive error instead.
no-panicRusthigh panic!/todo!/unimplemented!/unreachable! in non-test code — surface as an error so the app never crashes.
no-consoleTSlow A stray console.log(…) — remove before shipping.
tab-indentationanylow A tab in leading indentation — reformat to spaces.
missing-trailing-newlineanylow File doesn't end in a single \n — add one.
The audit is intentionally a heuristic over source text, not a compiler. It complements cargo fmt and clippy (which remain the hard gate) by catching the project-specific rules a generic linter doesn't know — and it does so deterministically, offline, in a few milliseconds.

CommandDoes
