A worker scans two barcodes — the spot, then the box. bOS checks the pair and turns the position green or red. Here is everything that must be in place.

How it works

Every scan — spot or box — lands in the same variable Devices\Scanner.Value. The Program tells them apart by their prefix (SPOT- vs BOX-), then matches the pair against the fixed pact Plats 1=BOX-A · 2=BOX-B · 3=BOX-C.

One thing to watch: duplicate scans

The Program fires on Devices\Scanner.Value OnChange. Scanning the same code twice in a row doesn’t change the value, so it won’t fire.

Fix (bOS side, no repo change): make the Program’s last command Set Value Devices\Scanner.Value = "". Every scan then starts from empty, so even a repeat counts as a change.

Setup order

  1. Enable RPC in bOS, set user/password, note the server IP.
  2. Build the logic + visual tiles from the co-pilot guide (incl. the = "" clear).
  3. Print & stick the SPOT / BOX labels.
  4. Point the scan app at Devices/Scanner · Value and whitelist the gun.
  5. Test Connection → Start Listening → scan SPOT-1 then BOX-A → tile goes green.

No gun handy? Type a value straight into Devices\Scanner.Value in the Configurator to simulate a scan.
